# Retrospectiva del equipo de desarrollo

## Contexto

Se realizó una retrospectiva del proceso de desarrollo de la landing page del Centro de Negocios Santiago de SERCOTEC, con el objetivo de identificar fortalezas, dificultades y oportunidades de mejora en el uso del framework, la gestión del proyecto y el trabajo colaborativo.

## Aspectos positivos

- Se logró construir una landing page moderna, clara e intuitiva.
- Se utilizaron componentes reutilizables en React.
- Se integró una API interna para administrar contenido dinámico.
- Se implementó un formulario con validación del lado del cliente y del servidor.
- Se documentó el uso del CMS mediante Postman.
- Se utilizó Git y GitHub para controlar versiones del proyecto.

## Dificultades encontradas

- Configuración inicial de Node.js, npm y políticas de ejecución en PowerShell.
- Integración entre frontend y backend.
- Organización inicial de carpetas y documentación.
- Comprensión del flujo de trabajo con Git, ramas y commits.

## Mejoras propuestas

- Implementar autenticación real para el panel administrativo.
- Agregar CAPTCHA al formulario de contacto.
- Optimizar imágenes usando formatos livianos como WebP.
- Desplegar el frontend y backend en servicios externos.
- Crear pruebas automatizadas para validar componentes y endpoints.
- Mejorar la experiencia móvil con un menú hamburguesa.

## Plan de acción para la próxima iteración

1. Crear un panel administrativo para gestionar servicios, testimonios y preguntas frecuentes.
2. Implementar autenticación para administradores.
3. Agregar protección avanzada contra bots en el formulario.
4. Desplegar el frontend en Vercel o Netlify.
5. Desplegar el backend en Render o Railway.
6. Incorporar pruebas automatizadas.
7. Mejorar la accesibilidad y navegación móvil.

## Conclusión

La retrospectiva permitió identificar que el proyecto cumple con los principales objetivos técnicos de la evaluación, especialmente en el uso de React, componentes reutilizables, consumo de API, validaciones, seguridad básica y documentación. Para una siguiente versión, se recomienda fortalecer el despliegue, la seguridad avanzada y la administración del contenido.
